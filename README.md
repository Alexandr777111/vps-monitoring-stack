# Lightweight VPS Monitoring Stack (Prometheus + Grafana + cAdvisor + Nginx)

Репозиторий содержит конфигурационные файлы и инфраструктурный код для развертывания трехуровневого стека мониторинга. Проект спроектирован и оптимизирован для работы в условиях **строго ограниченных ресурсов** (микро-VPS с 1 CPU и 1 ГБ RAM).

## 🏗️ Архитектура решения

Мониторинг охватывает всю экосистему сервера и разделен на три логических уровня:
1. **Уровень ОС и Железа (Host-level):** `Node Exporter` собирает низкоуровневые метрики Linux (нагрузка на CPU, утилизация RAM, дисковый ввод-вывод, сетевой трафик).
2. **Уровень Веб-сервера:** `Nginx` + `Nginx Prometheus Exporter` (порт 9113) для отслеживания активных соединений, RPS (запросов в секунду) и HTTP-статусов ответов.
3. **Уровень Контейнеризации (Docker-level):** `cAdvisor` инспектирует рантайм Docker и собирает индивидуальные метрики изолированных приложений (в качестве целевого сервиса развернута база данных `Redis:alpine`).

Все метрики агрегируются в базе данных **Prometheus** и визуализируются на кастомизированных аналитических панелях в **Grafana**.

## ⚙️ Ключевые оптимизации (Борьба за ресурсы)

В ходе развертывания были успешно решены проблемы острого дефицита RAM и дискового пространства микро-инстанса:
* **Тюнинг агента cAdvisor:** Добавлены флаги `--housekeeping_interval=15s` (снизил частоту опроса и утилизацию CPU процессом cAdvisor более чем в 10 раз) и `--docker_only=true` (полностью исключил дублирование системных метрик хоста и сэкономил оперативную память).
* **Глубокий траблшутинг диска (Troubleshooting):** С помощью конвейера утилит `du` и `sort` локализована критическая утилизация диска. Обнаружен и удален скрытый избыточный кэш сред разработки в директории суперпользователя. Это высвободило более 2 ГБ пространства, снизив показатель `Root FS Used` с опасных 96.7% до безопасных 74.3%.
* **Отказоустойчивость (Resilience):** Настроены правила автоматического перезапуска системных служб мониторинга через systemd (`Restart=always`, `RestartSec=5s`) для защиты от OOM-killer при пиковых нагрузках на память.
